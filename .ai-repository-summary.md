# ConvoContentBuddy Repository Summary

**AI-Powered Real-time Programming Interview Assistant**

## Repository Overview

This repository contains ConvoContentBuddy, an autonomous background listener designed to assist users during technical interviews and coding sessions. The system uses AI-powered speech recognition and semantic search to automatically identify algorithmic problems and provide optimal solutions.

## Architecture Overview

### Core Components

| Component | Type | Description |
|-----------|------|-------------|
| **ConvoContentBuddy.AppHost** | Aspire Orchestrator | Manages service lifecycle, health checks, and dependency injection |
| **ConvoContentBuddy.API.Brain** | ASP.NET Core Web API | Semantic Kernel hub with TMR (3 replicas), SignalR hub, and hybrid retrieval logic |
| **ConvoContentBuddy.UI.Web** | Blazor WebAssembly | Zero-interaction dashboard with live transcript and solution cards |
| **ConvoContentBuddy.Data.Seeder** | Worker Service | Ingests LeetCode problems and generates embeddings |
| **ConvoContentBuddy.ServiceDefaults** | Class Library | Shared OTLP configuration, health checks, and resilience policies |

### Infrastructure Dependencies

| Service | Purpose | Technology |
|---------|---------|------------|
| **Qdrant** | Vector similarity search | gRPC on port 6334 |
| **PostgreSQL** | Graph relationships and metadata | pgvector extension |
| **Redis** | SignalR backplane and caching | StackExchange.Redis |
| **Gemini API** | LLM inference and embeddings | Gemini 2.5 Flash, text-embedding-004 |

## Key Design Decisions

### 1. Triple Modular Redundancy (TMR)
- API.Brain runs with 3 replicas
- Redis backplane ensures SignalR connection persistence
- Automatic failover on instance failure

### 2. Hybrid Retrieval Strategy
- **Tier 1**: Qdrant vector search + PostgreSQL graph expansion + Gemini verification
- **Tier 2**: Fallback to secondary LLM provider
- **Tier 3**: Deterministic "Safe Mode" using local vectors only

### 3. Zero-Interaction UI
- Web Speech API for continuous transcription
- SignalR WebSocket for real-time updates
- Autonomous client-side buffer management
- Debounced POST requests to API

### Build & Test
```bash
# Build solution
dotnet build ConvoContentBuddy.slnx

# Run tests
dotnet test

# Run with coverage
dotnet test --collect:"XPlat Code Coverage"
```

### Local Development
```bash
# Start infrastructure
docker-compose up -d postgres qdrant redis

# Run API (with hot reload)
dotnet watch --project src/ConvoContentBuddy.API.Brain

# Run Blazor UI
dotnet watch --project src/ConvoContentBuddy.UI.Web

# Run Aspire orchestrator
dotnet run --project src/ConvoContentBuddy.AppHost
```
